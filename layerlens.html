Rethinking Layer Lists

LayerLens introduces two features that simplify navigation and reduce cognitive load in complex, collaborative files. This project explores how we can make design tools more intuitive for teams working on intricate interfaces.

My Role

As the lead UX designer, I was responsible for user research, concept development, and interface design. I collaborated closely with product managers and engineers to ensure the features aligned with user needs and technical constraints.

Problem

Design teams struggle with complex layer hierarchies in collaborative design files. The current layer list becomes overwhelming when multiple designers work on the same file, leading to:

  • Difficulty finding specific layers
  • Confusion about layer relationships
  • Reduced productivity in large files
  • Increased cognitive load for team members

User Research

We conducted interviews with 15 design teams and analyzed usage patterns from 1000+ design files. Key findings included:

78% of users struggle with layer organization
Average time to find a layer: 2.3 minutes
Teams with 5+ members report highest friction
Most requested: better search and filtering

LayerLens Features

Smart Grouping

Automatically groups related layers and provides quick access to common layer sets.

Contextual Search

Search layers by content, type, or relationship with intelligent suggestions.

Visual Hierarchy

Clear visual indicators show layer relationships and importance.

Quick Actions

Right-click context menu with common operations for faster workflow.
